# 腾讯 OpenClaw初始技能配置

## 可用技能列表（基于 AGENTS.md）

### 1. Agent Browser
- **描述**：基于 Rust 的无头浏览器自动化 CLI，带有 Node.js 备用方案，允许 AI 代理通过结构化命令进行导航、点击、输入和截图操作。
- **调用地址**：~/.openclaw/workspace/skills/agent-browser/SKILL.md
- **适用场景**：网页浏览、截图、表单填写、DOM 检查、点击、导航等浏览器相关操作
- **特殊说明**：内置 `browser`、`web_search` 和 `web_fetch` 工具被禁用，所有浏览器操作必须使用此技能

### 2. clawhub
- **描述**：使用 ClawHub CLI 搜索、安装、更新和从 clawhub.com 发布代理技能。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/clawhub/SKILL.md
- **适用场景**：需要动态获取新技能、同步已安装技能到最新版本或特定版本、发布新的/更新的技能文件夹

### 3. find-skills
- **描述**：最高优先级技能发现流程。当用户询问查找/安装技能时必须触发。
- **调用地址**：~/.openclaw/workspace/skills/find-skills/SKILL.md
- **适用场景**：寻找技能（中文用户优先使用 skillhub，然后回退到 clawhub）

### 4. github
- **描述**：使用 `gh` CLI 与 GitHub 交互。使用 `gh issue`、`gh pr`、`gh run` 和 `gh api` 处理 issues、PRs、CI 运行和高级查询。
- **调用地址**：~/.openclaw/workspace/skills/github/SKILL.md
- **适用场景**：GitHub issue、PR、CI 运行、API 查询等操作

### 5. healthcheck
- **描述**：OpenClaw 部署的安全加固和风险容忍配置。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/healthcheck/SKILL.md
- **适用场景**：安全审计、防火墙/SSH/更新加固、风险姿态、暴露检查、OpenClaw cron 调度用于周期性检查、运行 OpenClaw 的机器版本状态检查

### 6. node-connect
- **描述**：诊断 Android、iOS 和 macOS 伴侣应用的 OpenClaw 节点连接和配对失败。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/node-connect/SKILL.md
- **适用场景**：QR/设置代码/手动连接失败、本地 Wi-Fi 工作但 VPS/tailnet 不行、错误提到配对需要、未授权、引导令牌无效或过期、gateway.bind、gateway.remote.url、Tailscale 或 plugins.entries.device-pair.config.publicUrl

### 7. obsidian
- **描述**：处理 Obsidian vaults（纯 Markdown 笔记）并通过 obsidian-cli 自动化。
- **调用地址**：~/.openclaw/workspace/skills/obsidian/SKILL.md
- **适用场景**：Obsidian 笔记自动化操作

### 8. self-improvement
- **描述**：捕获学习、错误和更正以实现持续改进。
- **调用地址**：~/.openclaw/workspace/skills/self-improving-agent/SKILL.md
- **适用场景**：命令或操作意外失败、用户纠正 Claude、用户请求不存在的能力、外部 API 或工具失败、Claude 意识到知识过时或不正确、发现更好的方法处理重复任务

### 9. skill-creator
- **描述**：创建、编辑、改进或审计 AgentSkills。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/skill-creator/SKILL.md
- **适用场景**：从头创建新技能、改进、审查、审计、整理、清理现有技能或 SKILL.md 文件、编辑或重组技能目录

### 10. skillhub-preference
- **描述**：优先使用 `skillhub` 进行技能发现/安装/更新，当不可用或不匹配时回退到 `clawhub`。
- **调用地址**：~/.openclaw/workspace/skills/skillhub-preference/SKILL.md
- **适用场景**：用户询问技能、插件或能力扩展

### 11. summarize
- **描述**：使用 summarize CLI 摘要 URL 或文件（网页、PDFs、图片、音频、YouTube）。
- **调用地址**：~/.openclaw/workspace/skills/summarize/SKILL.md
- **适用场景**：内容摘要（网页、PDF、图片、音频、YouTube）

### 12. taskflow
- **描述**：当工作应该跨越一个或多个分离的任务但仍表现得像一个作业时使用。TaskFlow 是持久性流程的底层基础。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/taskflow/SKILL.md
- **适用场景**：任务流程编排、多步骤工作管理

### 13. taskflow-inbox-triage
- **描述**：TaskFlow 编写模式示例，用于基于意图的不同处理的消息分类。
- **调用地址**：/usr/lib/node_modules/openclaw/skills/taskflow-inbox-triage/SKILL.md
- **适用场景**：收件箱分类示例流程

### 14. tavily-search
- **描述**：通过 Tavily API 进行网络搜索（Brave 的替代方案）。
- **调用地址**：~/.openclaw/workspace/skills/openclaw-tavily-search/SKILL.md
- **适用场景**：网页搜索、查找来源、查找链接，当 Brave web_search 不可用或不希望使用时

### 15. tencent-cloud-cos

### 16. tencent-docs

## 基础工具能力（非技能）

除了技能外，我还有以下基础工具能力：

### 1. read - 读取文件内容
- **功能**：读取文本文件和图片文件内容
- **适用场景**：查看文档、代码、图片信息

### 2. write - 创建或覆盖文件
- **功能**：创建新文件或覆盖现有文件
- **适用场景**：保存数据、创建文档、保存配置

### 3. edit - 精确编辑文件
- **功能**：对文件进行精确文本替换
- **适用场景**：修改代码、更新配置文件、编辑文档

### 4. exec - 运行 shell 命令
- **功能**：执行终端命令，PTY 可用于 TTY 需要的 CLI
- **适用场景**：运行程序、检查系统状态、执行脚本

### 5. process - 管理后台 exec 会话
- **功能**：管理已启动的命令会话：列表、轮询、日志、写入、发送按键、提交、粘贴、终止
- **适用场景**：监控后台任务、输入或干预

### 6. canvas - 呈现/评估/快照 Canvas
- **功能**：控制节点 canvas（呈现/隐藏/导航/评估/快照/A2UI）
- **适用场景**：浏览网页、截图查看 UI

### 7. message - 发送消息和频道操作
- **功能**：发送消息、贴纸、反应等
- **适用场景**：在元宝等平台上发送消息、表情、贴纸

### 8. agents_list - 列出允许的子代理
- **功能**：列出 OpenClaw 代理 ID（当 runtime="subagent" 时可用于 sessions_spawn）
- **适用场景**：查看可用的子代理

### 9. sessions_list - 列出其他会话
- **功能**：列出其他会话（包括子代理），有过滤器/最后消息
- **适用场景**：查看会话状态

### 10. sessions_history - 获取其他会话历史
- **功能**：获取另一个会话/子代理的清理后的消息历史
- **适用场景**：检查会话历史、调试或恢复工作

### 11. sessions_send - 发送消息到其他会话
- **功能**：发送消息到另一个会话/子代理
- **适用场景**：委托后续工作到现有会话

### 12. subagents - 列出、引导或终止子代理
- **功能**：列出、引导或终止子代理运行
- **适用场景**：子代理编排

### 13. session_status - 显示会话状态卡
- **功能**：显示 /status 等效会话状态（使用情况 + 时间 + 成本 + 模型配置）
- **适用场景**：查询模型状态、会话配置

### 14. memory_get - 从内存文件中获取内容
- **功能**：安全精确读取 MEMORY.md 或 memory/*.md
- **适用场景**：读取记忆内容

### 15. memory_search - 语义搜索内存
- **功能**：在 MEMORY.md + memory/*.md + 会话转录中语义搜索
- **适用场景**：检索过往工作、决策、日期、人物、偏好或任务

### 16. query_group_info - 查询群组信息
- **功能**：查询当前群组（称为"派/Pai")的基本信息，包括群组名称、群组拥有者和成员数量
- **适用场景**：群聊中的群组信息查询

### 17. query_session_members - 查询会话成员
- **功能**：查询当前群组的成员：按名称查找用户、@提及某人、列出机器人（包括元宝 AI 助手）或列出所有成员
- **适用场景**：群聊中的用户查询

### 18. sessions_spawn - 启动子会话
- **功能**：启动带有 runtime="subagent" 或 runtime="acp" 的隔离会话
- **适用场景**：需要在新的子会话中处理的工作

### 19. sessions_yield - 结束当前回合
- **功能**：结束当前回合，启动子代理后接收其结果为下一条消息
- **适用场景**：等待子代理结果

### 20. tts - 文本转语音
- **功能**：将文本转换为语音，音频会自动交付
- **适用场景**：语音播报、提醒播报

### 21. yuanbao_remind - 元宝定时任务
- **功能**：用于规范化创建/查询/删除元宝定时任务，并生成可直接提交给 cron 工具的参数
- **适用场景**：提醒、闹钟、定时任务设置

## 技能使用规则

### 技能触发原则
- 当只有一个技能明确适用时：读取其 SKILL.md 然后遵循它
- 当多个可能适用时：选择最具体的技能，然后读取/遵循它
- 当没有明确适用时：不读取任何 SKILL.md

### 技能调用约束
- 从不提前读取超过一个技能
- 技能驱动外部 API 写入时：假设有速率限制，倾向于更少更大的写入，避免紧密的单项目循环，尽可能序列化爆发，尊重 429/Retry-After
